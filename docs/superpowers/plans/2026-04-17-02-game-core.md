> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Превратить `@hockey/game-core` из заглушки с одной константой в полный детерминированный движок броска: PRNG, геометрия катка, три паттерна движения вратаря, разрешение броска, каталоги вратарей и клюшек, формулы наград.
```ts
import seedrandom from 'seedrandom';

export interface Rng {
  next(): number;
  range(min: number, max: number): number;
}

export function createRng(seed: string): Rng {
  const gen = seedrandom(seed);
  return {
    next: () => gen(),
    range: (min, max) => min + Math.floor(gen() * (max - min)),
  };
}
```

- [ ] **Step 3: Реализация `goalie/patterns.ts`**

```ts
import type { GoalieConfig } from './types.js';
import type { Rng } from '../rng.js';
import type { Vec2 } from '../rink.js';
import { GOAL } from '../rink.js';

const goalCenterX = GOAL.x + GOAL.width / 2;
const goalY = GOAL.y + GOAL.height / 2;
const halfOpening = (GOAL.width / 2) * 0.9; // оставляем запас у штанг

function clampX(x: number): number {
  return Math.max(GOAL.x + 10, Math.min(GOAL.x + GOAL.width - 10, x));
}

/**
 * Линейный — вратарь едет от штанги к штанге с постоянной скоростью.
 * `speed` — полная ширина за 1/frequency секунд.
 */
export function linearPattern(cfg: GoalieConfig, _rng: Rng, t: number): Vec2 {
  const period = 1000 / Math.max(cfg.frequency, 0.1);
  const phase = ((t % period) / period) * 2 - 1; // -1..1
  const tri = phase < 0 ? -1 - 2 * phase : 1 - 2 * phase; // треугольная волна -1..1
  return { x: clampX(goalCenterX + halfOpening * cfg.amplitude * tri), y: goalY };
}

/**
 * Синусоида вокруг центра ворот.
 */
export function sinePattern(cfg: GoalieConfig, _rng: Rng, t: number): Vec2 {
  const omega = 2 * Math.PI * cfg.frequency / 1000;
  const x = goalCenterX + halfOpening * cfg.amplitude * Math.sin(omega * t);
  return { x: clampX(x), y: goalY };
}

/**
 * Рывки: каждые `1/frequency` секунд вратарь мгновенно «перелетает» в новую точку.
 * Между рывками стоит на месте. Точка выбирается детерминированно от (seed, step).
 */
export function dashPattern(cfg: GoalieConfig, rng: Rng, t: number): Vec2 {
  const period = 1000 / Math.max(cfg.frequency, 0.1);
  const step = Math.floor(t / period);
  // Используем rng «в фиксированном порядке»: шаг определяет сколько next()-ов
  // скипнуть. Это сохраняет детерминизм, т.к. rng получен из createRng(seed).
  let pick = 0;
  for (let i = 0; i <= step; i++) pick = rng.next();
  const offset = (pick * 2 - 1) * cfg.amplitude; // -amplitude..amplitude
  return { x: clampX(goalCenterX + halfOpening * offset), y: goalY };
}
```

> **Примечание:** `feint` оставляем на Plan 3 — для тренировочного режима не нужен (босс Финтарь недоступен без прогрессии). В `GoaliePatternId` тип есть, но реализации пока нет — бросаем ошибку при попытке использовать.

```ts
import type { GoalieConfig, GoalieState } from './types.js';
import { GOALIE_SIZE } from './types.js';
import { createRng } from '../rng.js';
import { linearPattern, sinePattern, dashPattern } from './patterns.js';

export function simulateGoalie(
  cfg: GoalieConfig,
  seed: string,
  shotIndex: number,
  t: number,
): GoalieState {
  // Каждый бросок — свой PRNG-поток.
  const rng = createRng(`${seed}:${shotIndex}:${cfg.id}`);
  let position;
  switch (cfg.pattern) {
    case 'linear':
      position = linearPattern(cfg, rng, t);
      break;
    case 'sine':
      position = sinePattern(cfg, rng, t);
      break;
    case 'dash':
      position = dashPattern(cfg, rng, t);
      break;
    case 'feint':
      throw new Error('feint pattern not implemented in Plan 2');
  }
  return { position, width: GOALIE_SIZE.width, height: GOALIE_SIZE.height };
}
```

```ts
import type { ShotInput } from './types.js';
import type { Vec2 } from '../rink.js';
import { PUCK_START, RINK } from '../rink.js';

export interface Trajectory {
  start: Vec2;
  end: Vec2;
  length: number;
  angle: number;
}

const MAX_RANGE = RINK.height * 1.5; // при power=1 шайба заведомо долетает

export function computeTrajectory(input: ShotInput): Trajectory {
  const length = Math.max(0, Math.min(1, input.power)) * MAX_RANGE;
  // angle = 0 → вверх (уменьшение y). Положительный angle → вправо.
  const dx = Math.sin(input.angle) * length;
  const dy = -Math.cos(input.angle) * length;
  return {
    start: { ...PUCK_START },
    end: { x: PUCK_START.x + dx, y: PUCK_START.y + dy },
    length,
    angle: input.angle,
  };
}
```

2. Пересекаем траекторию с AABB вратаря (в точке `releaseTime`) → `save`.
3. Пересекаем со штангами → `miss: wide`.
4. Пересекаем с проёмом ворот (GOAL_OPENING) → `goal`.
5. Иначе (траектория ушла мимо ворот вообще) → `miss: wide`.

Клюшка влияет через `shotZoneMultiplier`: AABB вратаря эффективно уменьшается на этот коэффициент (лучшая клюшка даёт больше «хорошей зоны» для обхода вратаря).

- [ ] **Step 2: Реализация `resolve.ts`**

Алгоритм — сегмент-AABB пересечение (Liang–Barsky или простой параметрический):

```ts
import type { ShotInput, ShotResult, StickEffects } from './types.js';
import type { GoalieState } from '../goalie/types.js';
import type { Vec2 } from '../rink.js';
import { GOAL, GOAL_OPENING } from '../rink.js';
import { computeTrajectory } from './trajectory.js';

const MIN_POWER = 0.1;

interface Aabb { x: number; y: number; width: number; height: number }

function segmentHitsAabb(
  start: Vec2,
  end: Vec2,
  box: Aabb,
): Vec2 | null {
  // Liang–Barsky
  const dx = end.x - start.x;
  const dy = end.y - start.y;
  let tMin = 0;
  let tMax = 1;
  const p = [-dx, dx, -dy, dy];
  const q = [start.x - box.x, box.x + box.width - start.x, start.y - box.y, box.y + box.height - start.y];
  for (let i = 0; i < 4; i++) {
    if (p[i] === 0) {
      if (q[i] < 0) return null;
    } else {
      const t = q[i] / p[i];
      if (p[i] < 0) { if (t > tMax) return null; if (t > tMin) tMin = t; }
      else          { if (t < tMin) return null; if (t < tMax) tMax = t; }
    }
  }
  return { x: start.x + dx * tMin, y: start.y + dy * tMin };
}

export function resolveShot(
  input: ShotInput,
  goalie: GoalieState,
  stick: StickEffects,
): ShotResult {
  if (input.power < MIN_POWER) return { type: 'miss', reason: 'short' };

  const tr = computeTrajectory(input);

  // 1) Пересечение с вратарём (AABB). Клюшка уменьшает эффективную ширину.
  const shrink = 1 / Math.max(stick.shotZoneMultiplier, 1);
  const effWidth = goalie.width * shrink;
  const goalieBox: Aabb = {
    x: goalie.position.x - effWidth / 2,
    y: goalie.position.y - goalie.height / 2,
    width: effWidth,
    height: goalie.height,
  };
  const savePoint = segmentHitsAabb(tr.start, tr.end, goalieBox);
  if (savePoint) return { type: 'save', goalieContact: savePoint };

  // 2) Штанги.
  if (segmentHitsAabb(tr.start, tr.end, GOAL.leftPost) ||
      segmentHitsAabb(tr.start, tr.end, GOAL.rightPost)) {
    return { type: 'miss', reason: 'wide' };
  }

  // 3) Пересекла ли траектория линию ворот в проёме?
  // Линия ворот — y = GOAL_OPENING.y. Если end.y > GOAL_OPENING.y — не долетела.
  if (tr.end.y > GOAL_OPENING.y) return { type: 'miss', reason: 'short' };

  const tGoalLine = (GOAL_OPENING.y - tr.start.y) / (tr.end.y - tr.start.y);
  const xAtGoal = tr.start.x + (tr.end.x - tr.start.x) * tGoalLine;
  if (xAtGoal < GOAL_OPENING.xMin || xAtGoal > GOAL_OPENING.xMax) {
    return { type: 'miss', reason: 'wide' };
  }

  // 4) Перелёт выше ворот.
  if (tr.end.y < GOAL.y - GOAL.height) {
    return { type: 'miss', reason: 'over' };
  }

  return { type: 'goal', hitPoint: { x: xAtGoal, y: GOAL_OPENING.y } };
}
```

```ts
import type { ShotResult } from '../shot/types.js';
import type { GoalieConfig } from '../goalie/types.js';
import type { Stick } from './sticks.js';

export function calcShotReward(
  result: ShotResult,
  goalie: GoalieConfig,
  stick: Stick,
  currentStreak: number,
): number {
  if (result.type !== 'goal') return 0;
  const streakBonus = 1 + Math.min(currentStreak * 0.1, 1.0);
  return Math.round(goalie.baseReward * stick.effects.rewardMultiplier * streakBonus);
}
```

## Риски и подводные камни

**PRNG-детерминизм между Node и браузером.** `seedrandom` использует `Math.imul` и простую арифметику — одинаково работает в V8 и в браузерных движках. Но если в будущем перейдём на WebCrypto или какой-то нативный RNG — детерминизм сломается. Проверка — запустить determinism.test в Node (vitest) и в `web` пакете после Plan 3 (через `@hockey/web` vitest + jsdom).

**Плавающая точка в `resolveShot`.** Пересечения считаются через double. Теоретически клиент и сервер могут разойтись в последнем бите, если процессоры/компиляторы разные. Для MVP — не боремся; если в Plan 4+ будут массовые 409 Conflict из-за последнего бита — введём `snap-to-grid` в координатах шайбы.

**`dashPattern` и порядок rng.next().** Функция проматывает rng вплоть до нужного шага. Если вызвать `dashPattern` с тем же rng дважды для разных t — поведение некорректно (rng уже «проеден»). Поэтому в `simulateGoalie` мы создаём **новый** rng на каждый вызов (`createRng(seed:shotIndex:id)`). Это конечно тратит ресурсы, но правильно.

**Тюнинг баланса.** Числа `speed/amplitude/frequency` в `GOALIES` взяты с потолка. После Plan 3 (когда увидим на экране) — пересмотрим в отдельной задаче; для этого пригодится storybook-style «playground» с ползунками, но это Plan 3+.
