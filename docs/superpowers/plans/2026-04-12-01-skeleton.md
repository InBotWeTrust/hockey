# Ultimate Hockey — План 1: Скелет

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Создать рабочий каркас монорепо с тремя пакетами (`game-core`, `server`, `web`), CI, Docker Compose, задеплоить заглушку на прод-VPS с реально доступным URL.

**Architecture:** pnpm workspaces, три пакета TypeScript. `server` — Fastify с `/health`. `web` — Vite + React с главным экраном-заглушкой. `game-core` — пустая библиотека, экспортирующая версию. Весь стек запускается через `docker compose up` локально и на VPS одним и тем же способом.

**Tech Stack:** pnpm 9, Node.js 20, TypeScript 5, Fastify 4, Vite 5, React 18, PostgreSQL 16, Redis 7, Caddy 2, Docker Compose, GitHub Actions, Vitest.

**Связанный спек:** [2026-04-12-ultimate-hockey-pwa-mvp-design.md](../specs/2026-04-12-ultimate-hockey-pwa-mvp-design.md)

**Результат плана:** открываешь `https://app.yourhockey.ru` — видишь главный экран React-приложения. Открываешь `https://app.yourhockey.ru/api/health` — получаешь `{"ok": true}`. В CI зелёные тесты.

## Предварительные условия

Перед началом плана должны быть:

1. Установлен **Node.js 20+** (`node --version`).
2. Установлен **pnpm 9+** (`pnpm --version`) — если нет: `npm i -g pnpm@9`.
3. Установлен **Docker Desktop** (локальная разработка) — `docker --version`, `docker compose version`.
4. Создан **GitHub-репозиторий** `InBotWeTrust/hockey`, локальный git уже инициализирован (спек закоммичен).
5. Есть **VPS** (Ubuntu 22.04, 2+ GB RAM, статический IP) с SSH-доступом по ключу.
6. Есть **домен** (например `yourhockey.ru`) с DNS A-записью на IP VPS.
7. Есть **Telegram-бот**, полученный у `@BotFather` (нужен только токен, настроим домен позже).
8. Есть **VK-приложение**, созданное в `vk.com/apps?act=manage` (нужны `app_id` и `secret`, redirect URI настроим позже).

Все значения секретов (токен бота, VK app secret, пароли БД) будут в `.env` — на локальной машине в `.env.local`, на проде через GitHub Secrets.

const schema = z.object({
  NODE_ENV: z.enum(['development', 'test', 'production']).default('development'),

  PORT: z.coerce.number().default(3000),

- [ ] **Step 7: Реализовать `app.ts`**

```typescript
import Fastify, { type FastifyInstance } from 'fastify';
import { healthRoutes } from './routes/health.js';
import { loadConfig, type AppConfig } from './config.js';

export interface BuildAppOptions {
  config?: AppConfig;
}

export async function buildApp(options: BuildAppOptions = {}): Promise<FastifyInstance> {
  const config = options.config ?? loadConfig();
  const app = Fastify({
    logger: {
      level: config.LOG_LEVEL,
      transport:
        config.NODE_ENV === 'development'
          ? { target: 'pino-pretty', options: { colorize: true } }
          : undefined,
    },
  });

  await app.register(healthRoutes);

  return app;
}
```

  workflow_dispatch:

  group: deploy-prod
  cancel-in-progress: false

jobs:
  deploy:
    runs-on: ubuntu-latest
    timeout-minutes: 30
    needs: []

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup SSH key
        run: |
          mkdir -p ~/.ssh
          echo "${{ secrets.DEPLOY_SSH_KEY }}" > ~/.ssh/id_ed25519
          chmod 600 ~/.ssh/id_ed25519
          ssh-keyscan -H ${{ secrets.DEPLOY_HOST }} >> ~/.ssh/known_hosts

      - name: Sync repository to server
        run: |
          rsync -az --delete \
            --exclude='.git' --exclude='node_modules' --exclude='.env' --exclude='.env.local' \
            -e "ssh -i ~/.ssh/id_ed25519" \
            ./ ${{ secrets.DEPLOY_USER }}@${{ secrets.DEPLOY_HOST }}:${{ secrets.DEPLOY_PATH }}/

      - name: Build and restart services
        run: |
          ssh -i ~/.ssh/id_ed25519 ${{ secrets.DEPLOY_USER }}@${{ secrets.DEPLOY_HOST }} "\
            cd ${{ secrets.DEPLOY_PATH }} && \
            docker compose pull || true && \
            docker compose build && \
            docker compose up -d && \
            docker image prune -f"

      - name: Smoke test

## Быстрый старт

**Требования:** Node.js 20+, pnpm 9+, Docker.

```bash
pnpm install
pnpm typecheck
pnpm test
pnpm --filter @hockey/game-core build
pnpm dev:server     # http://localhost:3000/health
pnpm dev:web        # http://localhost:5173
```

## Полный стек через Docker

```bash
cp .env.example .env
docker compose -f docker-compose.yml -f docker-compose.dev.yml build
docker compose -f docker-compose.yml -f docker-compose.dev.yml up -d
curl http://localhost:3000/health
```

## Команды

| Команда | Описание |
|---|---|
| `pnpm build` | Сборка всех пакетов |
| `pnpm test` | Прогон тестов во всех пакетах |
| `pnpm typecheck` | TypeScript проверка типов |
| `pnpm lint` | ESLint |
| `pnpm format` | Prettier auto-fix |
| `pnpm dev:server` | Dev-сервер `server` (watch) |
| `pnpm dev:web` | Dev-сервер `web` (vite) |

## Деплой

Main ветка деплоится автоматически в GitHub Actions (`.github/workflows/deploy.yml`). Ручной запуск через Actions → Deploy → Run workflow.
